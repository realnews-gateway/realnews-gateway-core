# Data Flow

The system follows a secure, anonymized, multi‑stage data flow:

1. **User Access**  
   - Entry via unified protocol layer  
   - Account login (identity separated from publishing)

2. **Content Submission**  
   - Anonymous upload  
   - Automatic encryption  
   - Metadata stripping

3. **Processing**  
   - Format normalization  
   - Hash‑based deduplication  
   - Routing to mirror nodes

4. **Storage**  
   - Decentralized persistence (IPFS / Arweave)  
   - Optional NGO/media endpoints

5. **Distribution**  
   - Public mirror sites  
   - Encrypted channels  
   - Offline bundles (optional)

All stages are designed to minimize traceability, resist censorship, and ensure long‑term availability.
